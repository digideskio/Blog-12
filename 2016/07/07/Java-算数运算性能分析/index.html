Java 算数运算性能分析

发表于 2016-07-07   |   分类于 Java   |  

前言

我们在Java中做算数运算的时候有多种选择：

  • num++;
  • num += 3;
  • num = num + 5;

那么这些写法有什么差异呢？这篇文章会就着编译后的结果进行简单分析。
如果你发现本文有任何错误或遗漏，欢迎在下方评论中批评指正，提前感谢喵~~

测试方法

  1. 安装JDK，并配置好环境变量。
  2. 在一个java文件中写好测试代码。
  3. 用javac编译测试代码。
  4. 用javap反编译class文件。

结论

本部分会随着文章的编写逐步更新，现有的结论可能不完整甚至完全错误，请等下面的TODO列表消失后再看这部分，在这之前请去上面看每一个部分的结论。

  1. 加减法运算的时候，++、--与+=、-=没有性能差异，都是比较快的，而num = num + xxx这种用法则比较慢。
  2. 乘除法以及取余的时候，两种写法并没有性能差异。
  • 通过本文使用的方法，你可以去做更多的测试，研究Java代码底层的实现原理。
  • 本文章的结论仅仅适用于解释运行模式。
  • 代码是给人读的，附带能给机器执行的功能，所以不要为了运行快那么一点，就写一些很难看懂的代码，那完全是得不偿失。
    而且Java在编译期的优化是非常少的，大部分优化都是在运行的时候动态做的，因此你辛辛苦苦做出的优化可能根本就没有用，甚至起负面作用也是可能的。
    实际上在开启Java的动态编译(默认开启)优化后，++、--是最快的，而后两种性能是相同的，因为这时候代码已经被进一步的编译为了本地机器码，而不再是Java虚拟机使用的代码。
